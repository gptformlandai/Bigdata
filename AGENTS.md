# Big Data Notes Authoring Instructions

The learner is a fresher to Big Data. For future phases, especially Big Data tools and platforms, write notes in baby steps.

## Style

- Start with simple language before professional terms.
- Explain what it is, why it exists, and what problem existed before it.
- Show where it fits in a real data pipeline.
- Include practical usage, commands, configs, code snippets, or examples when useful.
- Include real-world scenarios and system design implications.
- Explain trade-offs, failure modes, and common mistakes.
- Keep files modular by phase and topic.
- Avoid filler. Be detailed where the topic is a real Big Data tool or production pattern.
- For small supporting concepts, keep the explanation concise.

## Big Data Tool Topics Must Include

- beginner intuition
- definition
- why the tool/concept exists
- where it fits in the ecosystem
- how it works step by step
- how to use it practically
- real-world example
- system design considerations
- trade-offs
- failure modes
- common interview questions
- quick revision notes

## Preferred Learning Order Inside A Topic

1. Feel the idea with an analogy.
2. Define it technically.
3. Explain the problem it solves.
4. Walk through the flow.
5. Show a tiny example.
6. Connect it to production systems.
7. Discuss trade-offs and failures.
8. End with interview-ready language.
