Phase 0 builds the vocabulary and mental models required before Hadoop, Spark, Kafka, warehouses, lakehouses, and distributed systems.

The style here is intentionally lighter than the original 20-section prompt. Some topics need deep system design treatment, but many foundation topics only need clear intuition, practical examples, mistakes, and interview language.

## Phase Goal

By the end of this phase, you should be able to explain:

- how data is represented
- how files and formats store records
- how schemas keep systems aligned
- how Linux, networking, APIs, and SQL support data work
- how databases reason about indexes, transactions, ACID, CAP, consistency, latency, throughput, and scaling

## Suggested Study Flow

1. Read Topics 001-005 to understand data representation.
2. Read Topics 006-009 to understand data contracts.
3. Read Topics 010-013 to understand practical engineering basics.
4. Read Topics 014-020 slowly because these appear constantly in system design interviews.
5. Finish with `phase-00-review.md`.
